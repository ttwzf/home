# 工艺表格自动优化项目蓝图

## 1. 项目目标

本项目的目标是把工艺 Excel 表格中的重复材料信息自动整理到第一行材料名称中，减少人工复制、漏填和格式不一致的问题。

当前已经实现：

```text
可视化选择 Excel 文件
自定义触发关键词
自动识别材料名称、颜色、单位、单件用料列
按连续行规则追加材料信息
生成订单用量汇总工作表
订单用量汇总支持连续材料组/全表两种分支
保留原行、原公式、原格式
输出新的优化文件
防止重复追加
```

## 2. 当前项目结构

```text
工艺表格自动优化/
├─ visual_process_optimizer.py          可视化窗口程序
├─ optimize_process_table.py            核心优化逻辑
├─ optimize_rules.json                  规则配置文件
├─ create_template.py                   模板生成脚本
├─ 打开可视化工艺表格优化工具.bat        可视化工具启动入口
├─ 运行工艺表格自动优化工具.bat          简易选择文件启动入口
├─ 工艺表格自动优化模板.xlsx             使用模板
├─ 盘套工艺表.xlsx                       原始样表
└─ 盘套工艺表_已优化.xlsx                输出样例
```

## 3. 核心数据流程

```text
用户选择 Excel 文件
        ↓
读取 optimize_rules.json 触发关键词
        ↓
打开工作簿
        ↓
逐个工作表查找表头
        ↓
找到 材料名称 / 颜色 / 单位 / 单件用料 列
        ↓
逐行扫描材料名称
        ↓
判断第一行是否命中触发关键词
        ↓
向下收集连续行
        ↓
判断单位相同、单件用料相同
        ↓
拼接：材料名称*颜色
        ↓
写回第一行材料名称
        ↓
标绿、添加批注
        ↓
按 材料名称 / 颜色 / 单位 / 门幅规格 汇总订单用量
        ↓
新增 订单用量汇总 工作表
        ↓
另存为新的 Excel 文件
```

## 4. 当前核心规则

### 4.1 触发条件

第一行材料名称必须命中 `optimize_rules.json` 中的任一触发关键词。

默认：

```json
{
  "trigger_keywords": [
    "需复合"
  ]
}
```

如果 `trigger_keywords` 是空列表，则任意非空材料名称都可以作为第一行。

### 4.2 连续行判断

从第一行的下一行开始向下扫描。

后续行必须同时满足：

```text
材料名称非空
单位与第一行相同
单件用料与第一行相同
行号连续
```

遇到不满足条件的行，当前连续组结束。

### 4.3 拼接规则

后续连续行会被转换为：

```text
材料名称*颜色
```

多个后续行用 `+` 连接：

```text
第一行材料名称(材料A*颜色A+材料B*颜色B)
```

后续行材料名称中的 `（复合）` 会被删除。

### 4.4 防重复规则

如果第一行材料名称中已经包含本次将要追加的括号内容，工具会跳过该行，避免重复追加。

### 4.5 订单用量汇总规则

优化完成后，可以新增 `订单用量汇总` 工作表。

当前有两个汇总分支：

```text
默认分支：只汇总连续同单位同用量材料组
可选分支：全表汇总
```

汇总维度：

```text
材料名称
颜色
单位
门幅/规格
```

汇总指标：

```text
订单用量合计
```

当前会排除：

```text
材料名称为空
颜色为空
单位为空
订单用量为空
订单用量为 0
```

来源行号为可选列，用于复查汇总来源。

订单用量如果是简单公式，例如 `=F6*$G$5`，工具会解析单元格引用并计算出数值后汇总。

默认分支会复用材料优化规则来识别连续材料组：

```text
第一行材料名称命中触发关键词
后续行材料名称非空
单位相同
单件用料相同
行号连续
```

只有参与这些连续组的行会进入汇总表。

## 5. 关键代码说明

### 5.1 `optimize_process_table.py`

核心逻辑文件。

主要函数：

```text
load_trigger_keywords()
读取 optimize_rules.json 中的触发关键词。

find_headers()
自动查找表头行，以及材料名称、颜色、单位、单件用料所在列。

find_table_headers()
自动查找材料名称、颜色、单位、门幅/规格、单件用料、订单用量所在列。

base_target_material()
判断当前材料名称是否可以作为第一行触发。

simplify_next_material()
清理后续行材料名称，例如去掉“（复合）”。

optimize_workbook()
主处理函数，负责扫描工作簿、修改材料名称、保存结果。

add_order_usage_summary()
生成订单用量汇总工作表。

collect_matched_group_rows()
收集参与连续同单位同用量材料组的行号，供默认汇总分支使用。

build_order_usage_summary()
按分支生成订单用量汇总工作表。
```

后续如果增加新规则，优先修改 `optimize_workbook()` 周边逻辑。

### 5.2 `visual_process_optimizer.py`

可视化窗口文件。

主要职责：

```text
选择输入文件
设置输出文件
编辑触发关键词
调用 optimize_workbook()
展示处理结果
打开输出文件夹
控制是否生成订单用量汇总表
控制汇总表是否只统计连续同用量材料组
控制汇总表是否显示来源行号
```

该文件不直接实现表格规则，只负责界面和用户操作。

### 5.3 `optimize_rules.json`

配置文件。

当前只有：

```text
trigger_keywords
```

以后建议把更多可配置规则放到这里，例如：

```json
{
  "trigger_keywords": ["需复合"],
  "require_same_unit": true,
  "require_same_usage": true,
  "remove_words": ["（复合）"],
  "join_symbol": "+",
  "material_color_separator": "*",
  "create_summary": true,
  "include_source_rows": true,
  "summary_matched_groups_only": true
}
```

在真正写入 Excel 前，先生成一个预览列表：

```text

引用了哪些后续行
```

用户确认后再执行写入。

当前工具默认另存为新文件，已经比较安全。

后续可以增加：

```text
按日期时间命名
```

### 6.4 增加规则测试页

可视化工具中可以增加一个“规则测试”区域。

用户输入几行示例数据后，工具直接显示会不会命中、结果是什么。

### 6.5 支持批量处理

当前一次处理一个 Excel 文件。

后续可以增加：

```text

批量处理所有 .xlsx 文件
生成处理汇总表
```

### 6.6 生成处理报告

每次处理后生成一个报告文件：

```text

输入文件
输出文件

修改明细
跳过原因统计
```

这样方便追踪和复查。

### 6.7 汇总表增强

后续可以增加：

```text
按材料大类汇总
按颜色汇总
按单位汇总
按门幅规格筛选

```

可以增加跳过原因：


## 8. 维护注意事项

### 8.1 不要直接覆盖原文件

建议始终输出为：

```text
原文件名_已优化.xlsx
```

这样更安全。

### 8.2 表头名称是关键

工具依赖以下表头：

```text
材料名称
颜色
单位
单件用料
```

如果以后表头文字变化，需要同步更新 `find_headers()`。

### 8.3 注意中文编码

配置文件和说明文件建议保持 UTF-8 编码。

如果出现乱码，优先检查：

```text
optimize_rules.json
```

### 8.4 修改规则后要做复跑测试

每次改规则后建议至少验证：

```text
第一次运行能正确修改
第二次运行不会重复追加
单位不同不修改
单件用料不同不修改
关键词不命中不修改
```
